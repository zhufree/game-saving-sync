# 开发路线图

**项目方向**: 点对点游戏存档传输工具
**最后更新**: 2026-04-22

## 当前进度判断

## 新阶段规划

### 阶段一：项目重定位与配置迁移

**目标**: 删除旧同步概念，建立新配置模型。

交付物：
- `AppConfig` 支持 `steam_root`、`steam_libraries`、`snapshot_dir`、`transfer_history`。
- 保留旧配置兼容读取，允许从 `backup_dir` 迁移到 `snapshot_dir`。
- 日志和应用名称改为 Game Save Transfer。

### 阶段二：Steam 游戏库与存档发现

**目标**: 让用户配置 Steam 文件夹后，程序能找到候选游戏存档。

交付物：
- 解析 Steam libraryfolders。
- 扫描已安装游戏。
- 建立游戏存档规则模型。
- DST 作为第一个内置规则。
- 手动路径兜底。

### 阶段三：P2P 配对与传输原型

**目标**: 跑通“发送方生成密钥，接收方输入密钥，传输一个存档包”的闭环。

交付物：
- 配对密钥生成/解析。
- 会话状态机。
- 传输元数据和文件清单。
- 文件哈希校验。
- 传输层抽象，先允许本地/局域网实现，后续替换公网 P2P。

### 阶段四：安全快照与接收写入

**目标**: 降低覆盖存档风险。

交付物：
- 接收前快照。
- 目标路径预检。
- 游戏进程运行检测。
- 恢复快照能力。
- 传输历史记录。

### 阶段五：qfluentwidgets GUI

**目标**: 提供完整桌面客户端体验。

交付物：
- 首次启动向导。
- 首页/发送/接收/游戏库/设置页面。
- 传输进度和错误提示。
- Fluent 风格导航、卡片、按钮、对话框。

### 阶段六：测试与打包

**目标**: 形成可分发 MVP。

交付物：
- 单元测试和关键流程集成测试。
- PyInstaller 打包。
- Windows 10/11 验证。
- 许可说明和用户文档。

## 需要优先修改的文件

1. `src/config.py`: 替换旧同步字段。
2. `src/utils.py`: 保留通用函数，新增 Steam 路径和文件校验能力，移除 Syncthing 查找函数的核心地位。
3. `pyproject.toml`: 作为 uv 的唯一依赖源，包含 `PySide6-Fluent-Widgets`。
4. `src/ui/`: 新增 qfluentwidgets 页面结构。
5. `tests/test_config.py`: 跟随新配置模型更新。
6. `PHASE_2_P2P_TRANSFER.md`: P2P 连接与传输规划。
7. `PHASE_4_GUI.md`: 已改写为 qfluentwidgets GUI 规划。

## 近期建议里程碑

1. 先完成配置模型迁移和 Steam 目录扫描。
2. 再完成存档发现规则模型，至少支持 DST 一个游戏。
3. 然后做传输层 spike，验证真实 P2P 方案。
4. 最后再全面做 GUI，避免界面先绑定错误的后端抽象。
