# 阶段三：安全快照与接收写入

**目标**: 在接收存档并覆盖目标路径前，自动创建可恢复的本地快照，降低误覆盖风险。
**优先级**: P0
**前置依赖**: Steam/游戏存档发现、传输元数据。

## 设计变化

旧文档中的“备份管理系统”不再是核心产品卖点。新方向下，快照是接收流程的一部分：只要接收端即将写入已有存档，就必须先保护本地状态。

## 任务清单

### 3.1 快照创建

- [ ] 实现 `snapshot_manager.py`。
- [ ] 接收前检测目标路径是否存在。
- [ ] 若存在，默认创建 zip 快照。
- [ ] 快照名称包含游戏名、appid、时间戳。
- [ ] 记录快照元数据：源路径、目标路径、文件数量、大小、哈希。

### 3.2 写入预检

- [ ] 校验接收包元数据。
- [ ] 根据本地游戏规则计算目标路径。
- [ ] 展示覆盖范围和快照路径。
- [ ] 检测游戏进程是否运行。
- [ ] 权限不足时给出明确提示。

### 3.3 恢复能力

- [ ] 展示快照列表。
- [ ] 支持按游戏过滤。
- [ ] 支持恢复快照到原路径。
- [ ] 恢复前再次创建当前状态快照。

### 3.4 清理策略

- [ ] 支持按数量保留。
- [ ] 支持按天数保留。
- [ ] 默认不自动删除，避免误删救命快照。

## 验收标准

- [ ] 接收写入前会自动创建快照。
- [ ] 快照可以恢复。
- [ ] 游戏运行时默认阻止覆盖。
- [ ] 快照失败时不会继续覆盖存档。
- [ ] 所有风险提示能在 GUI 中清楚展示。
