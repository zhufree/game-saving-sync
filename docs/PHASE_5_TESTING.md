# 阶段五：测试与打包发布

**目标**: 验证 P2P 存档传输 MVP 的可靠性，并打包为 Windows 桌面应用。
**优先级**: P0
**前置依赖**: GUI、传输会话、安全快照完成。

## 测试范围

### 5.1 单元测试

- 配置模型迁移。
- Steam Library 解析。
- 游戏存档规则扫描。
- 配对密钥生成和解析。
- 存档包文件清单和哈希。
- 安全快照创建和恢复。

### 5.2 集成测试

- 发送方创建会话。
- 接收方输入密钥。
- 传输测试存档目录。
- 接收端创建快照并写入。
- 校验传输结果。
- 失败/断线/密钥过期流程。

### 5.3 GUI 测试

- 首次启动向导。
- Steam 路径选择。
- 发送流程。
- 接收流程。
- 覆盖确认对话框。
- 错误提示和重试。

### 5.4 Windows 兼容性

- Windows 10 64 位。
- Windows 11 64 位。
- Steam 默认安装路径。
- 多 Steam Library 路径。
- 非管理员用户权限。

## 打包

- 使用 PyInstaller。
- 包含 qfluentwidgets 资源。
- 打包为无控制台窗口应用。
- 输出安装包或便携 zip。

## 发布前检查

- [ ] README 与实际功能一致。
- [ ] 许可证说明包含 PySide6-Fluent-Widgets 的 GPLv3/商业授权风险。
- [ ] 首次启动不会产生无意义的旧配置字段。
- [ ] 没有强依赖 Syncthing。
- [ ] 没有默认覆盖用户存档的路径。

## 验收标准

- [ ] 新机器首次启动可完成 Steam 配置。
- [ ] 两个客户端可以完成一次存档传输。
- [ ] 接收端覆盖前快照可恢复。
- [ ] 打包后的 exe 可以运行 GUI。
- [ ] 关键流程有自动化或手动测试记录。
